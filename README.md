# EasyEnclave

EasyEnclave is a Rust control plane + agent system for deploying and attesting confidential compute workloads on [Intel TDX](https://www.intel.com/content/www/us/en/developer/tools/trust-domain-extensions/overview.html) VMs. It handles attestation verification, container orchestration, health monitoring, and secure ingress — so you can run workloads that prove they're running in a genuine trusted execution environment.

## Components

| Binary | Description |
|--------|-------------|
| `ee-cp` | Control-plane API server (Axum + SQLite/SQLx) |
| `ee-agent` | Node runtime — registers via TDX attestation, runs containers, reports health |
| `ee-admin` | Operator CLI (admin password hashing) |

## Architecture

```
                         +-----------------+
                         |    ee-cp        |
  Deployer/Admin ------> |  (Axum API)     |
                         |  SQLite + SQLx  |
                         +--------+--------+
                                  |
                    attestation   |   deploy / health
                    verification  |   commands
                                  |
              +-------------------+-------------------+
              |                                       |
      +-------+-------+                      +-------+-------+
      |   ee-agent    |                      |   ee-agent    |
      | (TDX VM)      |                      | (TDX VM)      |
      | Docker + OCI  |                      | Docker + OCI  |
      | Cloudflare    |                      | Cloudflare    |
      |   tunnel      |                      |   tunnel      |
      +---------------+                      +---------------+
```

**Agent registration flow:**
1. Agent requests nonce challenge from CP
2. Agent generates TDX quote via TSM
3. Agent calls Intel Trust Authority to get a signed attestation token
4. Agent sends token to CP — CP verifies Intel signature, nonce, and trusted measurement policy

**Control plane layering:** `routes/` (Axum handlers) → `services/` (domain logic) → `stores/` (SQLite via SQLx)

## API Overview

Key endpoints (full spec in `openapi/control-plane.yaml`):

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/health` | Health probe |
| `GET` | `/api/agents/challenge` | Get nonce for attestation |
| `POST` | `/api/agents/register` | Register agent with TDX quote |
| `POST` | `/api/deploy` | Deploy Docker Compose app |
| `GET` | `/api/agents` | List agents |
| `GET` | `/api/deployments` | List deployments |
| `POST` | `/api/accounts` | Create account |
| `POST` | `/api/admin/login` | Admin login |
| `GET` | `/api/stats/agents` | Agent statistics |
| `GET` | `/api/stats/apps` | App statistics |

## Local Development

Prerequisites:

- Rust stable toolchain
- `sqlite3` (optional but useful for inspection)
- Docker (only if you need local OCI runtime behavior)

Run checks:

```bash
cargo fmt --check
cargo clippy --workspace --all-targets -- -D warnings
cargo test --workspace
```

Run control plane locally:

```bash
CP_BIND_ADDR=0.0.0.0:8080 \
CP_DATABASE_URL='sqlite://easyenclave.db?mode=rwc' \
CP_ADMIN_PASSWORD='dev-password' \
cargo run --bin ee-cp
```

Health check:

```bash
curl -fsS http://127.0.0.1:8080/health | jq .
```

Run agent locally (example):

```bash
cat >/tmp/agent.json <<'JSON'
{
  "mode": "agent",
  "control_plane_url": "http://127.0.0.1:8080",
  "node_size": "tiny",
  "datacenter": "local:dev"
}
JSON

EASYENCLAVE_CONFIG=/tmp/agent.json cargo run --bin ee-agent
```

## Deploy with GitHub Actions

Add a workflow to your repo and push to `main` — that's it. No API keys, no config files.

```yaml
name: Deploy
on:
  push:
    branches: [main]

permissions:
  id-token: write    # GitHub OIDC — verified automatically, no secrets needed
  packages: write
  contents: read

env:
  IMAGE: ghcr.io/${{ github.repository }}:${{ github.sha }}

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - uses: docker/build-push-action@v6
        with:
          push: true
          tags: ${{ env.IMAGE }}

      - uses: devopsdefender/dd/.github/actions/deploy@main
        with:
          image: ${{ env.IMAGE }}
          app_name: my-app
          ports: "8000:8000"
```

The action handles OIDC authentication, deployment, and waits for your app to be live. See [`.github/actions/deploy/action.yml`](.github/actions/deploy/action.yml) for all available inputs (compose file, node size, datacenter, timeout, etc).

## Configuration

### Control Plane

Env-driven (`CP_*` prefix). Key vars:

| Variable | Default | Description |
|----------|---------|-------------|
| `CP_BIND_ADDR` | `0.0.0.0:8080` | Listen address |
| `CP_DATABASE_URL` | `sqlite://easyenclave.db?mode=rwc` | SQLite connection |
| `CP_ADMIN_PASSWORD` | — | Required for admin login flows |
| `CP_ATTESTATION_ALLOW_INSECURE` | `false` | Skip attestation verification (dev only) |

Additional optional vars for Cloudflare tunnels (`CLOUDFLARE_*`), Intel Trust Authority (`CP_ITA_*`), and GitHub OIDC (`CP_GITHUB_OIDC_*`). See [`src/cp/config.rs`](src/cp/config.rs) for the full list.

### Agent

JSON config at `/etc/easyenclave/agent.json` (override path with `EASYENCLAVE_CONFIG`):

```json
{
  "mode": "agent",
  "control_plane_url": "https://cp.example.com",
  "node_size": "tiny",
  "datacenter": "gcp:us-central1-a",
  "intel_api_key": "...",
  "provided_app": null
}
```

See [`docs/runbooks/run-agent.md`](docs/runbooks/run-agent.md) for operational guidance on running agents from bare TDX hosts.
